# Interview Response - First 30/60/90 Days (Senior Data Architect)

## First 30 days (Understand and align)
- Meet key stakeholders: Director of Architecture, EMR/HIE leads, clinical, finance, operations, security, and data teams.
- Review current-state architecture, integrations, data models, and reporting flows across on-prem and Azure/Fabric.
- Assess governance maturity: data quality, metadata, lineage, standards, privacy/security controls.
- Deliverables: current-state assessment, risk/issues log, and prioritized 90-day architecture backlog.

## Days 31-60 (Design and prove)
- Define target-state data architecture for HIE + financial/operational reporting (Fabric/OneLake, medallion, star/snowflake where needed).
- Publish core artefacts: conceptual/logical model, integration patterns, migration patterns, and architecture principles.
- Stand up governance working model: data ownership, stewardship, standards, and architecture review cadence.
- Deliverables: target architecture blueprint, standards pack, and 2-3 quick-win initiatives approved.

## Days 61-90 (Execute and embed)
- Start implementation of priority components (high-value integrations, critical data products, and quality controls).
- Run architecture governance forums and support delivery teams with design decisions and risk mitigation.
- Establish measurable KPIs: data quality, latency, interoperability reliability, and reporting performance.
- Deliverables: first production-ready architecture outcomes, roadmap for next 2 quarters, and handover cadence to BAU teams.

## Optional 60-second version
In my first 30 days, I would focus on stakeholder alignment and a clear current-state assessment across HIE, EMR, reporting, and governance controls. By day 60, I would have a target-state architecture and standards pack in place, plus a prioritized set of quick wins approved by governance. By day 90, I would aim to have initial architecture components in delivery, governance forums operating effectively, and measurable KPIs in place for data quality, interoperability reliability, and reporting performance.
